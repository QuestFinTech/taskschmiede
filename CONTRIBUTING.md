# Contributing to Taskschmiede

## Contribution model

Thank you for your interest in contributing to Taskschmiede.

This repository follows a maintainer-led contribution model:

- Anyone may view, clone, and fork this repository.
- External contributions are welcome via pull requests from forks.
- Direct push access to this repository is limited to maintainers.
- Maintainers review pull requests and decide whether they fit the roadmap, architecture, and quality standards of the project.

## How to contribute

1. Fork the repository
2. Create a feature branch in your fork
3. Make your changes
4. Open a pull request against `main`
5. Respond to review feedback if requested

## Notes

- Opening a pull request does not guarantee acceptance.
- Large changes should ideally be discussed first in an issue.
- Please keep pull requests focused and well described.
- Run `make lint` and `make test` before submitting.
- By submitting a contribution, you confirm that you have the right to license your work under the repository's open-source license.

## Maintainer policy

For security, stability, and architectural consistency, direct commits to protected branches are restricted to repository maintainers.
